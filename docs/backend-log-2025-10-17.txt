
Bricky — Технически дневник (Backend)

Дата: 2025-10-17

===============================
📅 ВЧЕРА (постигнато)
===============================
1. Инициализирахме backend проекта:
   - Инсталиран Node.js и npm
   - Инсталиран NestJS CLI глобално (npm install -g @nestjs/cli)
   - Създаден NestJS проект в bricky/backend
   - Тестов endpoint `/` върна “Hello World!” успешно

2. Оправена Git структура:
   - Свързан проектът с GitHub репозитори
   - Настроен `.gitignore`
   - Направен първи commit и push

3. Създаден NestJS модул `requests`:
   - Използван Nest CLI → `nest g resource requests`
   - Добавени файлове: controller, service, dto, entities, module
   - Импортиран в `app.module.ts`
   - Стартиран сървър без грешки

4. Създадено първо API (временна памет):
   - `POST /requests` → добавя нова заявка
   - `GET /requests` → връща всички заявки
   - Тест с Postman премина успешно

===============================
📅 ДНЕС (планирано)
===============================
1. Интеграция с база данни (PostgreSQL + TypeORM):
   - Инсталиране на PostgreSQL локално
   - Добавяне на TypeORM към NestJS (`npm install @nestjs/typeorm typeorm pg`)
   - Настройка на връзката в `app.module.ts`
   - Свързване на `Request` ентитито с таблица в базата

2. Персистентно съхранение на заявки:
   - `POST /requests` → записва в база
   - `GET /requests` → чете от база
   - Проверка, че данните се пазят след рестарт

3. (по избор, ако остане време)
   - Създаване на endpoint `DELETE /requests/:id`
   - Подготовка за frontend интеграция

💡 След днешния ден ще имаш реален, устойчив backend със съхранение в база данни — готов да се свърже с frontend приложението.
